# Choosing a Kernel

Compare the four built-in kernels and select the right one for your CAD workflow. This guide helps you decide based on language, export formats, and use cases.

## Prerequisites

- [Install @taucad/kernels](/docs/getting-started/installation)
- Completed the [Quick Start](/docs/getting-started/quick-start)

## Kernel Comparison

| Kernel | Language | Strengths | Export Formats | Use Cases |
|--------|----------|-----------|----------------|-----------|
| **Replicad** | TypeScript/JS | Parametric BRep via OpenCASCADE. STEP export. Strong typing. | glTF, GLB, STEP, STL, STL-binary | Professional CAD, mechanical parts, STEP interchange |
| **OpenSCAD** | OpenSCAD | CSG modeling. Familiar `.scad` syntax. Mature ecosystem. | glTF, GLB, STL, STL-binary | 3D printing, functional parts, CSG workflows |
| **JSCAD** | TypeScript/JS | Parametric CSG with `@jscad/modeling`. Browser-native. | glTF, GLB | Web-based CAD, parametric designs, lightweight models |
| **Zoo (KCL)** | KCL | Cloud-native CAD. AI integration via Zoo API. | glTF, GLB, STEP, STL, STL-binary | Cloud workflows, AI-assisted modeling, remote execution |

## Decision Criteria

### 1. Language and Ecosystem

- **TypeScript/JS**: Choose Replicad or JSCAD if you want to write CAD code in TypeScript. Replicad uses BRep (boundary representation); JSCAD uses CSG (constructive solid geometry).
- **OpenSCAD**: Choose OpenSCAD if you have existing `.scad` files or prefer the OpenSCAD language.
- **KCL**: Choose Zoo if you need cloud execution or AI integration via the Zoo API.

### 2. Export Requirements

- **STEP**: Required for professional CAD interchange. Use Replicad or Zoo.
- **STL for 3D printing**: All kernels except JSCAD support STL. Replicad, OpenSCAD, and Zoo support both ASCII and binary STL.
- **glTF/GLB for web viewers**: All kernels produce glTF. Use any kernel for web display.

### 3. Modeling Paradigm

- **BRep (Replicad)**: Best for precise mechanical parts, fillets, chamfers, and STEP export.
- **CSG (OpenSCAD, JSCAD)**: Best for boolean operations, 3D printing, and procedural designs.

## When to Use Each Kernel

### Replicad

Use Replicad when you need:

- STEP export for CAD interchange
- BRep modeling with fillets and chamfers
- TypeScript/JS with strong typing
- Professional mechanical design

```typescript
import { createKernelClient, fromMemoryFS } from '@taucad/kernels';
import { replicad } from '@taucad/kernels/kernels';
import { esbuild } from '@taucad/kernels/bundler';

const client = createKernelClient({
  kernels: [replicad()],
  bundlers: [esbuild()],
  fileSystem: fromMemoryFS(),
});
```

### OpenSCAD

Use OpenSCAD when you have:

- Existing `.scad` files
- CSG-based designs for 3D printing
- Familiarity with OpenSCAD syntax

```typescript
import { createKernelClient, fromMemoryFS } from '@taucad/kernels';
import { openscad } from '@taucad/kernels/kernels';

const client = createKernelClient({
  kernels: [openscad()],
  fileSystem: fromMemoryFS(),
});
```

### JSCAD

Use JSCAD when you need:

- Parametric CSG in TypeScript
- Lightweight browser-based CAD
- No STEP or STL export (glTF/GLB only)

```typescript
import { createKernelClient, fromMemoryFS } from '@taucad/kernels';
import { jscad } from '@taucad/kernels/kernels';
import { esbuild } from '@taucad/kernels/bundler';

const client = createKernelClient({
  kernels: [jscad()],
  bundlers: [esbuild()],
  fileSystem: fromMemoryFS(),
});
```

### Zoo (KCL)

Use Zoo when you need:

- Cloud execution via Zoo API
- AI-assisted modeling
- KCL language with remote engine

```typescript
import { createKernelClient, fromMemoryFS } from '@taucad/kernels';
import { zoo } from '@taucad/kernels/kernels';

const client = createKernelClient({
  kernels: [zoo({ baseUrl: 'wss://your-zoo-server/v1/kernels/zoo' })],
  fileSystem: fromMemoryFS(),
});
```

## Multi-Kernel Support

You can register multiple kernels in a single client. The framework selects the kernel automatically based on file extension and `canHandle` detection:

```typescript
import { createKernelClient, fromMemoryFS } from '@taucad/kernels';
import { replicad, openscad, jscad } from '@taucad/kernels/kernels';
import { esbuild } from '@taucad/kernels/bundler';

const client = createKernelClient({
  kernels: [replicad(), openscad(), jscad()],
  bundlers: [esbuild()],
  fileSystem: fromMemoryFS(),
});

// .ts/.js files -> Replicad or JSCAD (based on imports)
// .scad files -> OpenSCAD
const result = await client.render({ code: { 'model.ts': '...' } });
```

Kernel selection order follows the order in the `kernels` array. The first kernel whose `canHandle` returns true is used. See [Kernel Selection](/docs/concepts/kernel-selection) for details.

## Variations

- **Tau kernel**: Use the `tau()` kernel for importing CAD files (STEP, STL, OBJ, etc.) to glTF without writing code. It handles format conversion only.
- **Replicad options**: Pass `{ withExceptions: true }` to `replicad()` for detailed OpenCASCADE error messages during development.
- **Zoo options**: Zoo requires `baseUrl` for the WebSocket connection to the Zoo engine.
