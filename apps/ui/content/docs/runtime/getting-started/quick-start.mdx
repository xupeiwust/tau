# Quick Start

Render a 3D model from TypeScript code using the Replicad kernel. You will install the package, create a runtime client, and render geometry to glTF.

## Prerequisites

- Node.js 20 or later
- A package manager: npm, pnpm, or yarn

## Install

```bash tab="pnpm"
pnpm add @taucad/runtime
```

```bash tab="npm"
npm install @taucad/runtime
```

```bash tab="yarn"
yarn add @taucad/runtime
```

For framework-specific configuration (Vite, Node.js), see [Installation](./installation).

## Render a 3D Model

Create a file called `render.ts` and paste the following code:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());
const result = await client.export('glb', {
  code: {
    'main.ts': `
    import { drawRoundedRectangle } from 'replicad';
    export default () => drawRoundedRectangle(30, 50, 5).sketchOnPlane('XY').extrude(10);
  `,
  },
  file: 'main.ts',
});

if (!result.success) throw new Error(`Export failed: ${result.issues[0]?.message}`);
console.log(`Exported ${result.data.bytes.byteLength} bytes (${result.data.mimeType})`);
client.terminate();
```

`presets.all()` includes all built-in kernels, middleware, and bundlers. Inline `code:` input runs in an in-memory filesystem; the first command call starts the worker.

### How the runtime is wired

A `RuntimeClient` is composed of three orthogonal plug points:

| Plane          | What it owns                                                                    | Default in this snippet                |
| -------------- | ------------------------------------------------------------------------------- | -------------------------------------- |
| **Transport**  | The wire (`MessagePort` / in-process channel / Electron `MessagePortMain`)      | `inProcessTransport`                   |
| **Runner**     | Where kernel code executes (this isolate / `Worker` / `worker_thread` / remote) | in-process (same isolate)              |
| **Filesystem** | The source of truth for files (`fromMemoryFs`, `fromNodeFs`, `fromChannelFs`)   | inline memory FS, derived from `code:` |

The same three plug points compose every supported topology — browser UI, in-process Node, Electron renderer + main, multi-window Electron — without core changes. See [Worker Model → Topology Recipes](../concepts/worker-model#topology-recipes) for the canonical T1–T4 configs.

For tree-shaking, configure individual plugins with `createRuntimeClientOptions`:

```typescript
import { createRuntimeClient, createRuntimeClientOptions } from '@taucad/runtime';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const options = createRuntimeClientOptions({
  kernels: [replicad()],
  bundlers: [esbuild()],
});

const client = createRuntimeClient(options);
```

## Expected Output

```
Exported 5824 bytes (model/gltf-binary)
```

`result.data.bytes` is a `Uint8Array` containing a glTF binary (GLB) that you can write to disk, stream over HTTP, or hand to a 3D viewer like Three.js. The exact byte count depends on the model and tessellation settings.

> **Want live re-rendering?** This snippet runs a single render-and-export
> cycle. For UI viewers that need to react to parameter changes or file edits,
> see [Live Rendering](../guides/live-rendering) for the autonomous-loop API
> (`client.openFile`, `client.updateParameters`, `client.setOptions`, and the
> `geometry` event).
