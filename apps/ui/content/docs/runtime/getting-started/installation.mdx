# Installation

Install `@taucad/runtime` and its peer dependencies to start using the multi-kernel CAD runtime in your project.

## Prerequisites

- Node.js 20 or later
- A package manager: npm, pnpm, or yarn

## Install the Package

```bash tab="pnpm"
pnpm add @taucad/runtime
```

```bash tab="npm"
npm install @taucad/runtime
```

```bash tab="yarn"
yarn add @taucad/runtime
```

## Framework Configuration

### Vite

`@taucad/runtime` uses Web Workers for kernel isolation. Vite handles worker URLs natively via `new URL(..., import.meta.url)`:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());
```

### Node.js

For server-side usage, workers run as Node.js worker threads. Use `fromNodeFs` for filesystem access:

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { fromNodeFs } from '@taucad/runtime/filesystem/node';
import { inProcessTransport } from '@taucad/runtime/transport/in-process';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const client = createRuntimeClient({
  transport: inProcessTransport({ fileSystem: fromNodeFs('/path/to/project') }),
  kernels: [replicad()],
  bundlers: [esbuild()],
});
```

## Subpath Imports

The package exposes multiple subpath imports for tree-shaking:

| Import Path                            | Contents                                                                                                       |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| `@taucad/runtime`                      | Client, presets, `fromMemoryFs`, `fromFsLike`, plugin factories, error types, core types                       |
| `@taucad/runtime/kernels`              | Kernel factories (`replicad`, `opencascade`, `manifold`, `jscad`, `zoo`, `tau`)                                |
| `@taucad/openscad`                     | OpenSCAD kernel (separate package, GPL-2.0-or-later — install with `pnpm add @taucad/openscad` to opt in)      |
| `@taucad/runtime/kernel`               | Kernel-author entry (`defineKernel`, helpers, schemas) for building out-of-tree kernels                        |
| `@taucad/runtime/middleware`           | `defineMiddleware` and built-in middleware factories                                                           |
| `@taucad/runtime/bundler`              | `defineBundler` and esbuild bundler factory                                                                    |
| `@taucad/runtime/transport`            | Author API only (`defineRuntimeTransport`, `runtimeProtocolSchemas`, types)                                    |
| `@taucad/runtime/transport/in-process` | Same-isolate transport (`inProcessTransport`) — cross-environment                                              |
| `@taucad/runtime/transport/web`        | Browser-only transport (`webWorkerTransport`) — DOM `Worker` host                                              |
| `@taucad/runtime/transport/node`       | Node-only transport (`nodeWorkerTransport`) — gated so browser bundles don't externalize `node:worker_threads` |
| `@taucad/runtime/filesystem`           | `fromMemoryFs`, `fromFsLike`, `fromChannelFs`, `fromBrowserFs`                                                 |
| `@taucad/runtime/filesystem/node`      | `fromNodeFs` (Node-only)                                                                                       |
| `@taucad/runtime/filesystem/browser`   | Browser File System Access helpers where split from the main `filesystem` entry                                |
| `@taucad/runtime/types`                | All TypeScript type exports                                                                                    |
| `@taucad/runtime/testing`              | Testing utilities for kernel and middleware development                                                        |
| `@taucad/runtime/worker/web`           | Browser worker entry / bootstrap                                                                               |
| `@taucad/runtime/worker/node`          | Node worker entry / bootstrap                                                                                  |
| `@taucad/runtime/worker-internals`     | `KernelRuntimeWorker` ([worker model](../concepts/worker-model))                                               |

`fromMemoryFs` and `fromFsLike` are available from both the main `@taucad/runtime` entry and `@taucad/runtime/filesystem`. `fromNodeFs` is only available from `@taucad/runtime/filesystem/node` because it depends on Node.js APIs; `fromBrowserFs` is only available from `@taucad/runtime/filesystem/browser` because it depends on the browser-only File System Access API.
