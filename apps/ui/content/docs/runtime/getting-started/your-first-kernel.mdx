# Your First Kernel

Build a parametric box with rounded edges using the Replicad kernel. You will learn how to set up a runtime client, write parametric CAD code, render geometry, and export to STEP format.

## What You'll Learn

- How to configure a runtime client with [`createRuntimeClient`](../api/client)
- How to write parametric CAD models that accept user parameters
- How to render models and inspect the result
- How to export models to STEP format for professional CAD workflows

## Prerequisites

- [Install @taucad/runtime](./installation)
- Completed the [Quick Start](./quick-start)

## Step 1: Set Up the Client

Create a runtime client with the Replicad kernel and esbuild bundler:

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const client = createRuntimeClient({
  kernels: [replicad()],
  bundlers: [esbuild()],
});
```

`createRuntimeClient` returns a high-level client that manages worker lifecycle, kernel selection, and middleware. Inline `code:` input runs in an in-memory filesystem.

**Verify:** No output yet. The client initializes lazily -- the worker is only spawned on the first command call.

## Step 2: Write a Parametric Model

Define a CAD model that creates a box with configurable dimensions and fillet radius. Parameters make your model flexible -- users can adjust dimensions without editing code.

```typescript
const modelCode = `
  import { drawRoundedRectangle } from 'replicad';

  const defaultParams = {
    width: 30,
    height: 20,
    depth: 10,
    fillet: 3,
  };

  export default function main(params = defaultParams) {
    const { width, height, depth, fillet } = params;
    return drawRoundedRectangle(width, height, fillet)
      .sketchOnPlane('XY')
      .extrude(depth);
  }
`;
```

The `main` function is the entry point that the kernel calls. It receives a `params` object whose shape is inferred from `defaultParams`. The kernel extracts these parameters automatically and provides a JSON Schema for UI generation.

**Verify:** This step assigns a string -- no execution yet. Confirm that `modelCode` is a valid string with no syntax errors.

## Step 3: Render and Export to glTF

Hand the model code, entry file, and parameters to `client.export('glb', input)` -- this self-renders the model and returns the binary glTF in one call:

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const modelCode = `
  import { drawRoundedRectangle } from 'replicad';
  export default function main(params = { width: 30, height: 20, depth: 10, fillet: 3 }) {
    const { width, height, depth, fillet } = params;
    return drawRoundedRectangle(width, height, fillet).sketchOnPlane('XY').extrude(depth);
  }
`;

const client = createRuntimeClient({ kernels: [replicad()], bundlers: [esbuild()] });
const result = await client.export('glb', {
  code: { 'main.ts': modelCode },
  file: 'main.ts',
  parameters: { width: 50, depth: 15 },
});

if (result.success) {
  console.log(`Rendered ${result.data.bytes.byteLength} bytes (${result.data.mimeType})`);
} else {
  for (const issue of result.issues) {
    console.error(`[${issue.severity}] ${issue.message}`);
  }
}
```

`export(format, input)` self-renders the input and resolves a single
`ExportResult` -- the `success` discriminant narrows the payload so you can
safely access `result.data.bytes` in the `true` branch. If rendering fails, the
`issues` array contains structured errors with messages, locations, and
severity levels.

**Verify:** You should see output like `Rendered 5824 bytes (model/gltf-binary)`. If you see errors instead, check that your model code has correct Replicad imports and a default-exported `main` function.

> Need a UI viewer that re-renders on parameter changes or file edits? See
> [Live Rendering](../guides/live-rendering) for the autonomous-loop API
> (`client.openFile`, `client.updateParameters`, `client.setOptions`, and the
> `geometry` event).

## Step 4: Export to STEP

The same `export()` call swaps formats by changing the first argument -- the runtime selects the right export route based on the active kernel's capabilities:

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const modelCode = `
  import { drawRoundedRectangle } from 'replicad';
  export default function main(params = { width: 30, height: 20, depth: 10, fillet: 3 }) {
    const { width, height, depth, fillet } = params;
    return drawRoundedRectangle(width, height, fillet).sketchOnPlane('XY').extrude(depth);
  }
`;

const client = createRuntimeClient({ kernels: [replicad()], bundlers: [esbuild()] });
const stepResult = await client.export('step', {
  code: { 'main.ts': modelCode },
  file: 'main.ts',
  parameters: { width: 50, depth: 15 },
});

if (stepResult.success) {
  const { name, bytes, mimeType } = stepResult.data;
  console.log(`Exported: ${name} (${bytes.byteLength} bytes, ${mimeType})`);
} else {
  console.error('Export failed:', stepResult.issues);
}
```

STEP export is available for the Replicad and OpenCASCADE kernels. Other kernels support different export formats -- see [Choose a Kernel](../guides/choosing-a-kernel) for a comparison.

**Verify:** You should see output like `Exported: main.step (12345 bytes, model/step)`. The byte count varies with model complexity.

## Step 5: Clean Up

Terminate the client to shut down workers and release resources:

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const client = createRuntimeClient({ kernels: [replicad()], bundlers: [esbuild()] });
const result = await client.export('glb', {
  code: {
    'main.ts':
      "import { drawCircle } from 'replicad';\nexport default () => drawCircle(10).sketchOnPlane('XY').extrude(5);",
  },
  file: 'main.ts',
});
console.log(`Rendered ${result.success ? result.data.bytes.byteLength : 0} bytes`);

client.terminate();
```

**Verify:** The process should exit cleanly. If it hangs, `terminate()` was not called or a listener is keeping the event loop alive.

You built a parametric 3D model using the Replicad kernel:

1. Created a [`RuntimeClient`](../api/client) with `createRuntimeClient`
2. Wrote a parametric model with configurable dimensions
3. Rendered it to glTF with parameter overrides via `client.export('glb', input)`
4. Re-exported the same model to STEP by swapping the format argument
5. Used `result.success` to handle success and error cases
