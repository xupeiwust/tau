# Handle Errors

Kernel operations return a discriminated union `KernelResult<T>`. Check the `success` field to narrow results, and inspect `KernelIssue` for messages, severity, location, and stack frames. Each `KernelIssue` carries a stable `code: KernelIssueCode` field — branch on it instead of `RegExp`-matching `issue.message`. Lifecycle errors (`RuntimeNotConnectedError`, `RuntimeConnectionError`, `RuntimeTerminatedError`) and render errors (`RenderTimeoutError`, `RenderAbortedError`, `NoRenderOutcomeError`) signal exceptional flow that callers must handle as Promise rejections — every class exposes a stable `code` getter that is the canonical branch key.

## Prerequisites

- [Install @taucad/runtime](../getting-started/installation)
- Completed the [Quick Start](../getting-started/quick-start)

## Goal

Handle success and error results from `openFile`, `export`, and `getParameters` using the `success` discriminant and the `KernelIssue` structure. Handle render supersession via the `RenderOutcome` discriminated union, and surface typed lifecycle/render errors that arrive via Promise rejection.

## Steps

### 1. Understand the KernelResult Discriminated Union

`KernelResult<T>` is a union of two shapes:

```typescript
import type { KernelIssue } from '@taucad/runtime';

export type KernelSuccessResult<T> = {
  success: true;
  data: T;
  issues: KernelIssue[];
};

export type KernelErrorResult = {
  success: false;
  issues: KernelIssue[];
};

export type KernelResult<T> = KernelSuccessResult<T> | KernelErrorResult;
```

The `success` field is the discriminant. When `success` is `true`, `data` is present. When `success` is `false`, only `issues` is present.

### 2. Narrow Results with the `success` Field

The discriminated `KernelResult` shape is what `client.export(format, input)`
resolves with -- the `success` branch narrows `result.data` for you:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());

const result = await client.export('glb', {
  code: { 'main.ts': `export default function main() { return null; }` },
  file: 'main.ts',
});

if (result.success) {
  console.log(`Exported ${result.data.bytes.byteLength} bytes`);
} else {
  console.error('Export failed:', result.issues.map((i) => i.message).join('; '));
}
```

For UI viewers driving the autonomous render loop, the same `KernelResult`
shape arrives over the `geometry` event -- see [Live Rendering](./live-rendering)
for the subscription pattern.

### 3. Inspect KernelIssue Structure

Each `KernelIssue` has a message, severity, optional location, and optional stack information:

```typescript
import type { KernelIssue } from '@taucad/runtime';

const issue: KernelIssue = {
  message: 'Cannot read property "extrude" of undefined',
  code: 'RUNTIME',
  severity: 'error',
  type: 'runtime',
  location: {
    fileName: 'main.ts',
    startLineNumber: 5,
    startColumn: 10,
    endLineNumber: 5,
    endColumn: 20,
  },
  stackFrames: [
    {
      fileName: 'main.ts',
      functionName: 'main',
      lineNumber: 5,
      columnNumber: 10,
      context: 'user',
    },
  ],
};
```

| Field         | Type                                                                  | Purpose                                                          |
| ------------- | --------------------------------------------------------------------- | ---------------------------------------------------------------- |
| `message`     | `string`                                                              | Human-readable error message                                     |
| `severity`    | `'error' \| 'warning' \| 'info'`                                      | Issue severity                                                   |
| `type`        | `'compilation' \| 'runtime' \| 'kernel' \| 'connection' \| 'unknown'` | Error origin                                                     |
| `location`    | `ErrorLocation?`                                                      | File and range for editor integration                            |
| `stack`       | `string?`                                                             | Raw stack trace                                                  |
| `stackFrames` | `KernelStackFrame[]?`                                                 | Parsed frames with `context` (user, library, framework, runtime) |

### 4. Handle Errors in Export Calls

`client.export` resolves with a `KernelResult` whose `success` discriminant
narrows `result.data`. Iterate `issues` on the failure branch to surface
structured error information back to your UI or CLI:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const userCode = `
  import { drawCircle } from 'replicad';
  export default () => drawCircle(10).sketchOnPlane('XY').extrude(5);
`;

const client = createRuntimeClient(presets.all());
const result = await client.export('glb', {
  code: { 'main.ts': userCode },
  file: 'main.ts',
});

if (result.success) {
  const { bytes, name, mimeType } = result.data;
  const blob = new Blob([bytes], { type: mimeType });
  console.log(`Exported: ${name} (${blob.size} bytes)`);
} else {
  for (const issue of result.issues) {
    console.error(`[${issue.severity}] ${issue.message}`);
    if (issue.location) {
      console.error(`  at ${issue.location.fileName}:${issue.location.startLineNumber}`);
    }
  }
}
```

### 5. Handle Errors over the geometry Event Channel

For autonomous-loop UIs, the same `KernelResult` shape arrives on the
`geometry` event. Subscribe with `client.on('geometry', handler)` and
discriminate on `result.success` -- see [Live Rendering](./live-rendering) for
the full subscription lifecycle.

### 6. Handle Supersession via RenderOutcome

`openFile`, `updateParameters`, and `setOptions` return a `RenderOutcome`
discriminated union. Supersession is **not** an exception — it is a benign
outcome that indicates a newer call took ownership of the worker before this
one settled. The newer call's settlement carries the authoritative geometry.

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const code =
  "import { drawCircle } from 'replicad';\nexport default () => drawCircle(10).sketchOnPlane('XY').extrude(5);";

const client = createRuntimeClient(presets.all());
const settlement = await client.openFile({ code: { 'main.ts': code }, file: 'main.ts' });

if (settlement.superseded) {
  // Benign — no error; the latest geometry will arrive over `'geometry'`.
  console.log('Superseded by a newer call');
} else {
  // settlement.geometry is HashedGeometryResult — typed and narrowed.
  console.log(`Settled with ${settlement.geometry.success ? 'data' : 'issues'}`);
}
```

Real failures still throw via Promise rejection. Use the realm-safe type
guards rather than `instanceof` for cross-realm checks:

| Error                          | Trigger                                                                                                                                                                                    | `code`                              | Type guard                       |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------- | -------------------------------- |
| `RuntimeNotConnectedError`     | `FileInput` command issued before `connect()` resolved.                                                                                                                                    | `'RUNTIME_NOT_CONNECTED'`           | `isRuntimeNotConnectedError`     |
| `RuntimeConnectionError`       | Transport handshake failed. Carries `causeKind: 'transport-open' \| 'capabilities-resolution' \| 'kernel-binding'`.                                                                        | `'RUNTIME_CONNECTION_FAILED'`       | `isRuntimeConnectionError`       |
| `RuntimeTerminatedError`       | Command issued after `terminate()`, or in-flight Promise rejected because `terminate()` happened mid-flight. Carries `causeKind: 'explicit' \| 'connection-failed' \| 'transport-closed'`. | `'RUNTIME_TERMINATED'`              | `isRuntimeTerminatedError`       |
| `RenderTimeoutError`           | Render exceeded the configured `renderTimeout` (ms).                                                                                                                                       | `'RUNTIME_RENDER_TIMEOUT'`          | `isRenderTimeoutError`           |
| `RenderAbortedError`           | Render aborted (typically because a newer command superseded it).                                                                                                                          | `'RUNTIME_RENDER_ABORTED'`          | `isRenderAbortedError`           |
| `NoRenderOutcomeError`         | `client.export(format)` called with no prior render.                                                                                                                                       | `'RUNTIME_NO_RENDER_OUTCOME'`       | `isNoRenderOutcomeError`         |
| `SharedPoolEntryNotFoundError` | Worker emitted a pooled geometry key the runtime client could not resolve.                                                                                                                 | `'RUNTIME_SHARED_POOL_KEY_MISSING'` | `isSharedPoolEntryNotFoundError` |

Branch on `error.code` (or use the realm-safe type guard) — never `Error.message`. Messages are human prose that may change; codes are the contract.

> `RuntimeNotConnectedError` is thrown when a `FileInput` command runs against
> a client whose transport has no filesystem bridge. Pass a filesystem via the
> transport at construction (`inProcessTransport({ fileSystem })` or `webWorkerTransport({ fileSystem })`). See
> [Embedding in a Host](./embedding-in-a-host) for host-wiring patterns.

```typescript
import { createRuntimeClient, presets, isRenderTimeoutError, isRuntimeTerminatedError } from '@taucad/runtime';

const code =
  "import { drawCircle } from 'replicad';\nexport default () => drawCircle(10).sketchOnPlane('XY').extrude(5);";

const client = createRuntimeClient(presets.all());

try {
  await client.openFile({ code: { 'main.ts': code }, file: 'main.ts' });
} catch (error) {
  if (isRenderTimeoutError(error)) {
    console.warn('Render exceeded its timeout budget');
  } else if (isRuntimeTerminatedError(error)) {
    console.warn('Client was terminated while a command was in flight');
  } else {
    throw error;
  }
}
```

### 7. Error Enrichment (Kernel Authors)

Kernels enrich raw errors into `KernelIssue` with location and stack. Replicad maps OpenCASCADE exceptions; JSCAD and Zoo map their engine errors. When building custom kernels, use `createKernelError`:

```typescript
import { createKernelError } from '@taucad/runtime';

function validateWidth(width: number) {
  if (width > 0) return undefined;
  return createKernelError([
    {
      message: 'Invalid parameter: width must be positive',
      code: 'RUNTIME',
      severity: 'error',
      type: 'runtime',
      location: { fileName: 'main.ts', startLineNumber: 3, startColumn: 5 },
    },
  ]);
}

const issue = validateWidth(-1);
console.log(issue?.success === false ? issue.issues[0]?.message : 'ok');
```

## Variations

- **Display in UI**: Use `issue.location` to highlight errors in a code editor (e.g., Monaco markers).
- **Filter by severity**: Iterate `result.issues` and handle `error` vs `warning` differently.
- **Success with warnings**: `KernelSuccessResult` can have non-empty `issues` (e.g., deprecation warnings). Check `result.issues` after confirming success.
- **Connection errors**: `type: 'connection'` indicates transport/worker failures rather than kernel logic.
- **Autonomous mode errors**: Subscribe to `client.on('state', (state, detail) => ...)` to receive error state transitions in autonomous rendering mode.
