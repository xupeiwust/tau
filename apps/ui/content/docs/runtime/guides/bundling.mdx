# Bundling

`@taucad/runtime` ships dozens of static and dynamic assets — kernel WASM blobs, font atlases, transcoder plugins, middleware modules. Every consumer needs the same handful of bundler invariants for those assets to resolve at runtime. The runtime ships a one-line plugin per supported bundler so you never copy these rules into your config.

## TL;DR

```typescript
import { runtime } from '@taucad/runtime/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [runtime()],
});
```

That single plugin:

- Sets the [cross-origin-isolation](./cross-origin-isolation) headers on dev and preview servers.
- Excludes the runtime and every WASM-bearing dependency from Vite's dependency pre-bundling.
- Keeps `.wasm` files out of the base64 inline path.
- Forces `worker.format: 'es'` so worker bundles preserve `import.meta.url`.

If any of these go missing, the runtime appears to load successfully and then 404s on the first WASM fetch — usually under a stale-looking dependency hash.

## Bundler matrix

| Bundler              | Plugin                             | Notes                                                                                      |
| -------------------- | ---------------------------------- | ------------------------------------------------------------------------------------------ |
| Vite 7+ / Vite 8     | `@taucad/runtime/vite#runtime`     | Wraps the COI middleware automatically. Pass `{ crossOriginIsolation: false }` to opt out. |
| Rolldown 1.0+        | `@taucad/runtime/rolldown#runtime` | Externalises the runtime + every WASM-bearing dep; forces ESM output.                      |
| Webpack 5 / Parcel 2 | _(not shipped)_                    | The same invariants are required; track the issue in the research doc below.               |

## How asset emission works

Every WASM, font, and dynamically-imported plugin chunk inside `@taucad/runtime` is referenced via the **static `new URL(literal, import.meta.url)`** pattern:

```typescript
const wasmUrl = new URL('./wasm/replicad.wasm', import.meta.url).href;
```

This pattern is recognised by Vite, Rolldown, Webpack 5, Parcel 2, and esbuild as a first-class **asset reference**: at build time, the bundler copies the file to your `dist/`, hashes its filename, and rewrites the literal to the hashed URL. Any deviation — a variable, a function call, a template literal with an `${expression}` — opts out of asset emission, and the file silently fails to ship beside the bundle.

The runtime enforces this invariant via the workspace `tau-lint/static-import-meta-url` rule, scoped to `packages/runtime/src/**` and `kernels/openscad/src/**`. Consumers do not need to enable the rule themselves.

## Dynamic plugin loading

Kernels, middleware, and transcoders are loaded by `await import(moduleUrl)` against URLs produced by the same `new URL` pattern. Each plugin therefore needs its own chunk in the consumer bundle.

The runtime's `tsdown.config.ts` already declares one entry per plugin module, so each one ships as a discrete file in the published `dist/`. The Vite/Rolldown plugins exclude `@taucad/runtime` from pre-bundling so the published chunks reach your build untouched and `import.meta.url` continues to point at the correct asset directory at runtime.

## Common failure modes

| Symptom                                                                        | Likely cause                                                                                                                                                       |
| ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `404` on `node_modules/.vite/deps/.../wasm/...` after first dev render         | Pre-bundling rewrote the kernel; missing `optimizeDeps.exclude`. Use `runtime()`.                                                                                  |
| WASM blob inlined as a base64 data URL; worker fails on `instantiateStreaming` | Asset inlined under default 4 KB threshold or by a custom rule. `runtime()` opts WASM out.                                                                         |
| `Cannot find module ./middleware/parameter-cache.js`                           | Bundler aggressively folded the runtime into one chunk. Externalise it (the Rolldown plugin does this automatically).                                              |
| `crossOriginIsolated === false` in DevTools, `SharedArrayBuffer` undefined     | Missing COI headers; either keep `runtime({ crossOriginIsolation: true })` or set them at the host layer (see [Cross-Origin Isolation](./cross-origin-isolation)). |

## Opting out

When the host already takes care of one of the invariants (e.g. an Express middleware that sets COOP/COEP, or a custom `optimizeDeps` config), pass an option to skip just that piece:

```typescript
import { runtime } from '@taucad/runtime/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [runtime({ crossOriginIsolation: false })],
});
```

The COI plugin remains exported separately (`crossOriginIsolation()`) for host stacks that compose multiple isolation layers — see the [Cross-Origin Isolation](./cross-origin-isolation) guide.

## Rolldown / `tsdown` consumers

Library authors building on top of `@taucad/runtime` with Rolldown (or `tsdown`) should mark the runtime and its WASM deps as external so the assets are resolved from `node_modules` at runtime instead of being copied into the library bundle:

```typescript
import { runtime } from '@taucad/runtime/rolldown';
import { defineConfig } from 'rolldown';

export default defineConfig({
  plugins: [runtime()],
});
```

The `@taucad/cli` package ships a thin shim built exactly this way: it bundles the CLI argument parser and command wiring (~3 KB) but leaves `@taucad/runtime` and every WASM-bearing dep external, deferring the kernels to npm.
