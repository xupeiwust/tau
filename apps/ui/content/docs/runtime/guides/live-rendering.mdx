# Live Rendering

Use `client.openFile`, `client.updateParameters`, `client.setOptions`, and the `geometry` event when you need a UI viewer that re-renders on parameter changes, file edits, or option updates. The runtime owns the render loop -- you push intent (open this file, update these parameters) and consume settled geometry over the event channel.

> For a single render-and-export cycle, use [`client.export(format, input)`](../api/client) (see the [Quick Start](../getting-started/quick-start)).

## When to Use This

The autonomous loop is the right tool when:

- A user can drag a slider and you want the viewport to redraw on every settled value
- A file watcher detects a save and you want a fresh render without re-issuing the bundle
- A toggle changes per-render kernel options (e.g. `renderTimeout`, tessellation tolerance) and the next render needs to pick them up
- Multiple producers can race to update the same target -- the runtime serialises them through a discriminated `RenderOutcome` union so you can detect superseded calls

If your call site never re-renders the same model twice in a row, prefer the one-shot `client.export(format, input)` overload -- it does not require the event subscription described below.

## Subscribe to the `geometry` Event

`client.on('geometry', handler)` returns an unsubscribe function. The handler is invoked once per render result with a [`HashedGeometryResult`](../api/types) -- a discriminated `KernelResult` whose `success: true` branch carries the array of geometries and whose `success: false` branch carries structured `KernelIssue`s.

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());

const unsubscribe = client.on('geometry', (result) => {
  if (result.success) {
    console.log(`Rendered ${result.data.length} geometries`);
    // hand result.data to your viewer
  } else {
    for (const issue of result.issues) {
      console.error(`[${issue.severity}] ${issue.message}`);
    }
  }
});
```

Always retain the returned unsubscribe function and call it when your component unmounts or your worker session ends -- the handler closes over your viewer, so leaking it leaks the viewer.

## Open a File for Autonomous Rendering

`client.openFile(input)` hands the worker a `(file, parameters)` pair and starts the render loop. The Promise it returns settles with a `RenderOutcome` discriminated union:

- `{ superseded: false, geometry }` -- this call's render produced the geometry now visible.
- `{ superseded: true }` -- a newer `openFile` / `updateParameters` / `setOptions` call took ownership before this one settled. The latest call's geometry will arrive over the event channel.

Supersession is **not** an error. It is the normal outcome whenever a user drags a slider faster than the kernel can re-mesh.

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());

const modelCode = `
  import { drawRoundedRectangle } from 'replicad';
  export default function main(params = { width: 30, depth: 10 }) {
    return drawRoundedRectangle(params.width, 50, 5)
      .sketchOnPlane('XY')
      .extrude(params.depth);
  }
`;

const settlement = await client.openFile({
  code: { 'main.ts': modelCode },
  file: 'main.ts',
  parameters: { width: 50, depth: 15 },
});

if (settlement.superseded) {
  // A newer openFile/updateParameters call took ownership before this one
  // settled -- ignore this Promise's geometry; the latest one is on its way
  // through the `geometry` event subscription.
  console.log('Superseded');
} else {
  console.log(`Rendered ${settlement.geometry.success ? 'ok' : 'failed'}`);
}
```

Inline `code:` input runs in an in-memory filesystem. For filesystem-backed inputs (the `FileInput` overload), see [Embedding in a Host](./embedding-in-a-host).

## Live Parameter Updates

Once a file is open, push parameter changes through `client.updateParameters(...)`. The worker debounces, re-renders, and emits the result on the `geometry` event. The Promise returns the same `RenderOutcome` shape as `openFile`, so a slider drag that fires three updates in 16ms produces two superseded settlements and one settled geometry:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());

await client.openFile({
  code: {
    'main.ts':
      "import { drawCircle } from 'replicad';\nexport default ({ radius = 10 } = {}) => drawCircle(radius).sketchOnPlane('XY').extrude(5);",
  },
  file: 'main.ts',
  parameters: { radius: 10 },
});

async function onSliderChange(width: number, depth: number) {
  const settlement = await client.updateParameters({ width, depth });
  if (settlement.superseded) {
    return; // a newer drag value already won
  }
  if (settlement.geometry.success) {
    console.log(`Settled at width=${width}, depth=${depth}`);
  }
}

await onSliderChange(50, 15);
```

`updateParameters` requires the client to be connected -- the Promise rejects with `RuntimeNotConnectedError` when issued before `connect()` (or `createRuntimeClient({ fileSystem })`/inline `code:` auto-connect) settles. See [Handle Errors](./error-handling) for the full taxonomy.

## Live Option Updates

Per-render kernel options (e.g. `renderTimeout`, tessellation tolerances when the active kernel exposes them as render options) are replaced via `client.setOptions(...)`. Like `updateParameters`, the call awaits the next render's settlement:

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());

await client.openFile({
  code: {
    'main.ts':
      "import { drawCircle } from 'replicad';\nexport default () => drawCircle(10).sketchOnPlane('XY').extrude(5);",
  },
  file: 'main.ts',
});

const settlement = await client.setOptions({ renderTimeout: 60 });
if (!settlement.superseded && settlement.geometry.success) {
  console.log('New options applied to the next render');
}
```

Use this when a UI toggle changes how the kernel meshes or times out -- not for one-off export options, which belong on the `client.export(format, input)` call directly.

## Cleanup

Tear down the autonomous loop in two steps: unsubscribe your `geometry` listener, then terminate the client to shut down the worker.

```typescript
import { createRuntimeClient, presets } from '@taucad/runtime';

const client = createRuntimeClient(presets.all());
const unsubscribe = client.on('geometry', (result) => {
  if (result.success) console.log(`Geometry: ${result.data.length}`);
});

// ...later, on component unmount:
unsubscribe();
client.terminate();
```

`client.terminate()` is idempotent and rejects every subsequent command with `RuntimeTerminatedError`. After termination, `client.lifecycleState === 'terminated'`.
