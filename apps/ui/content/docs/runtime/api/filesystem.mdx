# Filesystem API

Use the filesystem API when configuring how kernels read and write project files. The package provides constructors for Node.js, in-memory, and any fs-compatible backends, plus bridge utilities for cross-worker filesystem access.

## Constructors

| Function                        | Import Path                       | Description                                               |
| ------------------------------- | --------------------------------- | --------------------------------------------------------- |
| `fromNodeFs(basePath)`          | `@taucad/runtime/filesystem/node` | Node.js filesystem rooted at `basePath`                   |
| `fromMemoryFs(files?)`          | `@taucad/runtime`                 | In-memory Map-backed filesystem, optionally seeded        |
| `fromFsLike(fs, rootPath?)`     | `@taucad/runtime`                 | Any fs-compatible object (BrowserFS, memfs, polyfills)    |
| `createRuntimeFileSystem(base)` | `@taucad/runtime/filesystem`      | Wrap `RuntimeFileSystemBase` with enhanced helper methods |

## Bridge Utilities

For cross-worker filesystem access, these utilities create MessagePort-based bridges:

| Function                                       | Description                                                                                                                                                                                    |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `createBridgePort(handlers)`                   | Serve handlers in-process; returns **`BridgePort`** with a raw **`MessagePort`** for `postMessage(..., [port])` transfer to a callee worker.                                                   |
| `createBridgeServer(handlers, port, options?)` | Serve an object's methods over a `Port` adapter (`wrapMessagePort` when using bare `MessagePort`).                                                                                             |
| `createBridgeProxy<T>(port)`                   | Proxy client over a **`Port`** (never a bare transferred **`MessagePort`** without **`wrapMessagePort`**).                                                                                     |
| `createBridgeCall(port, method, args)`         | Lower-level RPC triple over a **`Port`**.                                                                                                                                                      |
| `exposeFileSystem(handlers, options?)`         | Worker-side listener for incoming bridge ports (pairs with **`createFileSystemBridge`**).                                                                                                      |
| `createFileSystemBridge(worker, options?)`     | Client side: opens a **`MessageChannel`**, transfers one end to an FS-owning worker. Returns **`FileSystemBridge`** with a wrapped **`Port`** for **`createBridgeProxy`** in the same isolate. |
| `catchMessages(port, handler)`                 | Buffer messages during initialization.                                                                                                                                                         |
| `extractTransferables(value)`                  | Collect **`ArrayBuffer`** transferables for **`postMessage`**.                                                                                                                                 |

## Usage

### Node.js Filesystem

```typescript
import { createRuntimeClient } from '@taucad/runtime';
import { fromNodeFs } from '@taucad/runtime/filesystem/node';
import { inProcessTransport } from '@taucad/runtime/transport/in-process';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const client = createRuntimeClient({
  transport: inProcessTransport({ fileSystem: fromNodeFs('/path/to/project') }),
  kernels: [replicad()],
  bundlers: [esbuild()],
});
```

### In-Memory Filesystem

```typescript
import { createRuntimeClient, fromMemoryFs } from '@taucad/runtime';
import { inProcessTransport } from '@taucad/runtime/transport/in-process';
import { replicad } from '@taucad/runtime/kernels';
import { esbuild } from '@taucad/runtime/bundler';

const client = createRuntimeClient({
  transport: inProcessTransport({
    fileSystem: fromMemoryFs({
      'main.ts': 'export default function main() { return null; }',
    }),
  }),
  kernels: [replicad()],
  bundlers: [esbuild()],
});
```

### Filesystem Bridge

```typescript
import { createBridgePort } from '@taucad/runtime/transport-internals';
import { fromNodeFs } from '@taucad/runtime/filesystem/node';

const fileSystem = fromNodeFs('/path/to/project');
const { port, dispose } = createBridgePort(fileSystem);

const worker = new Worker(new URL('./worker.ts', import.meta.url), { type: 'module' });
worker.postMessage({ fsPort: port }, [port]);

const cleanup = () => {
  worker.terminate();
  dispose();
};
```
